# win下安装 解压版mysql-5.7.19

1，解压后新建my.ini文件( 安装的时候只有basedir、datadir这两个配置项就可以 )

[Client]
port = 3306											#设置3306端口


[mysqld]
port = 3306											#设置3306端口
basedir = C:\mysql-5.7.19-winx64					# 设置mysql的安装目录
datadir = C:\mysql-5.7.19-winx64\data             	# 设置mysql数据库的数据的存放目录
max_connections=200									# 允许最大连接数
log_error = C:\mysql-5.7.19-winx64\mysql-error.log	
character-set-server=utf8
default-storage-engine=INNODB						# 创建新表时将使用的默认存储引擎
sql_mode=NO_ENGINE_SUBSTITUTION,STRICT_TRANS_TABLES


[mysql]
default-character-set=utf8							# 设置mysql客户端默认字符集

2，安装、卸载mysql服务

参考链接：https://blog.csdn.net/muyimo/article/details/69384740

安装服务

  1. 安装服务

    mysqld -install

要在bin目录下手动初始化data目录:
    mysqld --initialize-insecure              # root帐号不带密码
    mysqld --initialize                #自动生成带随机密码的root用户
    mysqld --initialize --console      #不加参数--console，就要在my.ini中定义log_error，里面会记录生成的临时密码

​

  • 获取初次安装时生成的临时密码：

    解决如下5.7.11增强了安全性，在安装的过程中会生产root的随机密码，先查看这个密码：
cat /var/log/mysqld.log | grep temporary password
---2016-06-03T16:49:41.645458Z 1 [Note] A temporary password is generated for root@localhost: n,LuI)Fz8,a3

然后运行 mysql_secure_installation进行安全安装输入上面得到的随机密码 n,LuI)Fz8,a3以后，修改输入新密码即可，注意要满足强度要求。
  1. 启动服务

​ net start mysql

​ mysql -uroot -p 'xx' #xx为刚才mysqld --initialize --console初始密码

  1. 更改mysql的root密码

    • 方式一

      进入cmd界面（不是mysql试图）使用：

mysqladmin -u'root' password  提示框中输入密码‘123456’

mysqladmin -u'username' -h HOSTNAME  password  'new_pass' -h

    • 方式二

      mysql> set password for root@localhost = password('123456');    # 这个好行不需要flush privileges;

    • 方式三

      mysql> update user set authentication_string=password('123456') where user='root';
mysql> flush privileges;

    • 方式四（忘记是否可行了）

      ALTER USER 'root'@'localhost' IDENTIFIED BY 'new_password';

    • 这种方式为啥不行呢？？？

      mysql> update mysql.user set password=password('881222') where user='root';
							Password=PASSWORD('new_pass')            这种么？？？

net stop mysql mysqld -remove #mysqld remove mysql2

# linux源码编译安装

cd /usr/local/src/mysql-5.7.19/

cmake -DCMAKE_INSTALL_PREFIX=/usr/local/mysql-5.7.19 -DMYSQL_DATADIR=/data/mysql-5.7.19 -DMYSQL_UNIX_ADDR=/data/mysql-5.7.19/mysql.sock -DWITH_INNOBASE_STORAGE_ENGINE=1 -DENABLED_LOCAL_INFILE=1  -DDEFAULT_CHARSET=utf8  -DDEFALUT_CHARSETS=all  -DWITH_BLACKHOLE_STORAGE_ENGINE=1  -DWITH_ARCHIVE_STORAGE_ENGINE=1  -DWITH_PARTITION_STORAGE_ENGINE=1   -DDEFAULT_COLLATION=utf8_general_ci  -DMYSQL_USER=mysql   -DEXTRA_CHARSETS=all  -DMYSQL_TCP_PORT=3306  -DWITH_DEBUG=0 -DWITH_BOOST=../boost_1_59_0

新加的参数：
从MySQL 5.7.5开始Boost库是必需的
-DWITH_BOOST=boost/boost_1_59_0/  指定boost的位置，如-DWITH_BOOST=../boost_1_59_0
-DDOWNLOAD_BOOST=1 -DWITH_BOOST=1    系统会在编译之前先下载boost

如果之前编译不成功，请执行以下命令：

make clean

rm CMakeCache.txt

创建user、group，修改datadir的属主和属组

(可选)安装ssl,可指定文件生成路径，默认为DATA里面, /usr/local/mysql-5.7.19/bin]#./mysql_ssl_rsa_setup

初始化数据库：/usr/local/mysql-5.7.19/bin]# ./mysqld --initialize --user=mysql --basedir=/usr/local/mysql-5.7.19 --datadir=/data/mysql-5.7.19

复制启动文件：/usr/local/mysql-5.7.19/support-files]#cp mysql.server /etc/init.d/mysqld 2，/etc/init.d/mysqld start 如果启动文件没有复制/etc/init.d/mysqld，启动方式如下： ./mysqld_safe --defaults-file=/etc/my.cnf &
