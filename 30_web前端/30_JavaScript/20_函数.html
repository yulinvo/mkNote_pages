# 形参、实参

function sum(a,b){ console.log(a+b) }

  • 实参比形参多时，多余参数会被忽略

    #解析器也不会检查实参的数量,多余实参不会被赋值

  • 实参比形参少时，没有传递的形参会被赋值为undefined

    sum(123);    --> NaN            #少的实参会被赋予undefined

# return返回值

return;         #相当于return undefined;
如果函数中不写return，则也会返回undefined

var result = alert("hello");        #此时result为undefined，证明alert是没有返回值的

使用return可以结束整个函数，不管return是不是写在主函数中，还是写在嵌套函数中！！！

  • throw也可中断函数执行，效果同return，示例nodejs中读取文件

    fs.readFile('./resources/为学.md', (err, data)=>{
    //如果失败, 则抛出错误
    if(err) throw err;									//效果同return
    //如果没有出错, 则输出内容
    console.log(data.toString());
});

# 函数作用域

函数作用域下的声明提前

在函数作用域中，使用var定义的变量会在函数中所有的代码执行之前被声明，不使用var声明的变量都会成为全局变量

有点类似与 let a=10 && a=20 两条命令，一个是定义在全局一个是定义在函数中，不过也不完全一样，参考示例二就没有事先定义变量d

  • 示例一

    var a=10;
function fun3(){
	console.log(a);                     #此时报undefined
	var a = 35;                            #如果代码是 a = 35上条语句打印的结果就是10，但是将全局的a修改了
}                                       

console.log(a);             #如果代码是var a = 35; 此处输出10
#如果代码是a = 35;此处输出的是35；

  • 示例二

    function fun5(){
	d = 100;                         //没有全局变量d的情况下
	//d没有使用var关键字，则会设置为全局变量，相当于window.d=100
}
fun5()
console.log(d);                 	//此时输出100

定义形参就相当于在函数作用域中声明了变量

#fun(e){}就相当于在函数内部定义，var e；

没有传递参数时直接是undefined (就算全局有变量e，全局的变量e和此处的e实际没有任何关系，此时的e只是一个标识可以是abc、test、e都行)

总结：函数内部声明的变量一定要写var

# call和apply

func.call()、func.apply()、func() 都会调用函数执行

call方法

let fn1 = function(){
    console.log(this);
}

  • 是函数.call()，不是对象.call()

    fn1.call({name:"aaa"})  不是{name:"aaa"}.call(fn1())

  • 是fn1.call({name:"aaa"})，不是fn1().call({name:"aaa"})，调用时不加()

  • call传递的参数可以是列表、对象、字符串、也可以是window

    fn1.call([10,20])
fn1.call({age:20})
fn1.call("sss")
fn1.call(window)        #fn1.call()不传递参数默认应该就是window吧？？？

call示例二

function fn1() {  console.log(this.stus);}      //this.stus
const banji = {
    name: "终极一班",
    stus: ['xiaoming','xiaoning','xiaotian','knight'],
}

fn1.call({ name: "tom",age:20,stus:"aa" })
fn1.call(banji)

call和apply的区别

call()方法可以将实参在对象之后依次传递                         #fun.call(obj,2,3);

apply()方法需要将实参封装到一个数组中统一传递         #fun.apply(obj,[2,3]);

# arguments

在调用函数所传递的实参都会在arguments类数组对象中保存（不管是否定义了形参），使用arguments[1]这种也能读取类数组对象中的第2个参数

因为arguments是一个隐藏的参数，而不是写在形参接收的位置，所以写不写形参都不影响argumets最终的结果；但是写在形参位置的reset参数就不是了（reset只保存未被接收的参数）

let f1 = function (a,b,c,d,e) {        //不管是否定义形参输出都是一样的
    console.log(arguments);      //[10, 20, 30, callee: ƒ, Symbol(Symbol.iterator): ƒ]
    console.log(arguments.length);
    console.log(...arguments,[...arguments]);
    
    console.log(arguments.callee == f1);   //callee属性
}

f1()
f1(10, 20)
f1(10, 20, 30)

callee属性 ：这个属性对应一个函数对象，就是当前正在指向的函数的对象

function fun(a,b){
   console.log(arguments.callee == fun)
}

# 闭包

保存多个a值，就是产生了多个闭包

function fn1(){
  var a = 2;
  function fn2(){   //执行函数定义就会产生闭包(不用调用内部函数)
    a++;
    console.log(a)
  }
  // fn2()
  return fn2
}

// fn1()
fn1()()         //3
fn1()()         //3
fn1()()         //3

var f = fn1()           //闭包函数只产生了一次，产生两次就再次执行下var f2 = fn1()
f()             //3
f()             //4
f()             //5

function fn1() {
  var a = 2;
  function fn2() {
    a++;
    console.log(a)
  }
  function fn3() {
    a--;
    console.log(a)
  }
  return {
    fn2,
    fn3
  }
}

var f = fn1()
console.log(f.fn3())

保存多个 i 值，就是产生了多个闭包
