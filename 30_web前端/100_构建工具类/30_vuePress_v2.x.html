# 快速上手

  1. 创建并进入一个目录
mkdir vuepress-starter && cd vuepress-starter

  1. 初始化

    git init
yarn init

  2. 本地安装 vuePress

    yarn add -D vuepress@nex

  3. 在 package.json添加一些script

    {
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}

  4. 将默认的临时目录和缓存目录添加到 .gitignore 文件中

    echo 'node_modules' >> .gitignore
echo '.temp' >> .gitignore
echo '.cache' >> .gitignore

  5. 创建你第一篇文档

    mkdir docs && echo '# Hello VuePress' > docs/README.md

  6. 在本地服务器上启动文档站点

    yarn docs:dev

# 配置

# 配置文件

docs目录下创建一个.vitepress的目录，必要的配置文件是config.js，目录解构如下：

├─ docs
│  ├─ .vuepress
│  │  └─ config.js
│  └─ README.md
├─ .gitignore
└─ package.json

config.js应当导出一个 JavaScript 对象： 【注意此配置项， 新版配置手册与旧版相比有更新】

export default {
  themeConfig: {
  	Nav: [...],
  	
  }
}

# 客户端配置文件

在大多数情况下，配置文件已经足够帮助你配置好你的 VuePress 站点。不过，有些时候用户们可能希望直接添加一些客户端代码。 VuePress 通过客户端配置文件来支持这种需求：

├─ docs
│  ├─ .vuepress
│  │  ├─ client.js   <--- 客户端配置文件
│  │  └─ config.js   <--- 配置文件
│  └─ README.md
├─ .gitignore
└─ package.json

需要注意的是，客户端配置文件需要使用 ESM 格式：

import { defineClientConfig } from '@vuepress/client'

export default defineClientConfig({
  enhance({ app, router, siteData }) {},
  setup() {},
  rootComponents: [],
})

# 页面

# 路由

默认配置下， README.md 和 index.md 都会被转换成 index.html ，并且其对应的路由路径都是由斜杠结尾的。然而，如果你想同时保留这两个文件，就可能会造成冲突。

在这种情况下，你可以设置 pagePatterns 来避免某个文件被 VuePress 处理，例如使用 ['/*.md', '!/README.md', '!.vuepress', '!node_modules'] 来排除所有的 README.md 文件。

# Frontmatter

Markdown 文件可以包含一个 YAML (opens new window) Frontmatter。你可以通过 Frontmatter 来覆盖当前页面的 lang, title, description 等属性。因此，你可以把 Frontmatter 当作页面级作用域的配置。

---
lang: zh-CN
title: 页面的标题
description: 页面的描述
---

提示

前往 Frontmatter 参考 (opens new window) 查看 VuePress 支持的 Frontmatter 配置。

前往 默认主题 > Frontmatter 参考 (opens new window) 查看默认主题的 Frontmatter 配置。

# 内容

页面的主要内容是使用 Markdown 书写的。VuePress 首先会将 Markdown 转换为 HTML ，然后将 HTML 作为 Vue 单文件组件的 <template> 。

借助 markdown-it (opens new window) 和 Vue 模板语法的能力，基础的 Markdown 可以得到很多的扩展功能。接下来，前往 Markdown (opens new window) 章节来了解 VuePress 中 Markdown 的扩展功能。

# 静态资源

# 相对路径

你可以在你的 Markdown 内容中使用相对路径来引用静态资源：

![图片](./image.png)

一般情况下，我们推荐你使用这种方式来引用图片，因为人们通常会把图片放在引用它的 Markdown 文件附近。

# Public 文件

你可以把一些静态资源放在 Public 目录中，它们会被复制到最终生成的网站的根目录下。默认的 Public 目录是 .vuepress/public ，可以通过 public (opens new window) 配置项来修改。

└─ docs
   ├─ .vuepress
   |  └─ public
   |     └─ images
   |        └─ hero.png  # <- Logo 文件
   └─ guide
      └─ assets.md       # <- 我们在这里			使用：			![VuePress Logo](/images/hero.png)

# Base Helper

如果你的网站部署在非根路径下，例如 https://foo.github.io/bar/ ，那么你应该把 base (opens new window) 设置为 '/bar/'。显然，此时你的 Public 文件会被部署在 https://foo.github.io/bar/images/hero.png 这样的链接下。

在大多数情况下，你不需要担心这些 Public 文件的引用路径，因为 VuePress 会自动帮你处理 base 前缀：

<!-- 你不需要给 `/images/hero.png` 手动添加 `/bar/` 前缀 -->
![VuePress Logo](/images/hero.png)

    { text: 'Home', link: '/' },

		text: 'Base基础技能',						//未下拉时即显示，而不是下拉后显示的内容
		children: [

            	text: 'SubGroup',				//不写text虽然不会报错，但是也会占用位置，导航条上会有一块空白的地方
                children: [
                    '/group/sub/foo.md',
                    '/group/sub/bar.md',
                    { text: 'Section A Item A', link: '...' },
                    { text: 'Section B Item B', link: '...' }
                ]
            }
		]
	},
