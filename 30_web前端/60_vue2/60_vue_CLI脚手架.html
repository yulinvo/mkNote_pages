# 脚手架的使用

  1. 安装

    npm install -g @vue/cli        
#安装后查看vue --version 或者 vue -V        #安装后就可以在任意目录下打开cmd直接输入Vue使用了

  2. 使用 切换到你要创建项目的目录，然后使用命令创建项目

    vue  create  xxx

  3. 启动项目

    npm run serve

  • 脚手架升级：npm i -g @vue/cli

  • vue 2.x项目创建的另一种方式（不推荐，只是记录下，还是推荐使用上面的方法创建）

    cnpm install -g @vue/cli-init

vue init webpack my-project
	#cnpm install -g webpack安装失败，重新安装最新的node版本(node-v14.15.1-x64.msi)之后，在安装成功
    
    build、config都是webpack的配置文件

cd my-project && npm install  && npm run dev（或者npm start）启动服务器

# 创建项目遇见的问题

  • vue init webpack vueproject时一直 downloading template，报错vue-cli · Failed to download repo vuejs-templates/webpack: connect ETIMEDOUT 192.30.253.112:443

    参考文档：https://www.jianshu.com/p/650fe2976fb9
实际解决方法：
    1，https://github.com/vuejs-templates/webpack地址将代码download zip（因为git clone需要做全局网络代理）
    2，解压文件到D:\code\web\base\webpack\webpack-develop
    3，vue init  D:\code\web\base\webpack\webpack-develop   vueproject即可

  • Vetur can’t find package.json in /xxxx/xxxxxx.

    使用命令cnpm -- install 代替 cnpm install来执行

# 配置npm加速

  • 方法一：通过命令配置

    npm config set registry https://registry.npm.taobao.org
#验证命令：npm config get registry    （默认是：https://registry.npmjs.org/ ）

  • 方法二：通过使用cnpm安装

    安装cnpm：    npm install -g cnpm --registry=https://registry.npm.taobao.org
使用cnpm：    cnpm install -g @vue/cli

# 脚手架目录结构

App.vue

<template>
  <div>我是App根组件</div>
</template>

<script>
export default {
  name:'',
}
</script>

<style>

</style>

main.js

import Vue from 'vue'
import App from './App.vue'

Vue.config.productionTip = false

new Vue({
  render: h => h(App),
}).$mount('#app')

public文件夹：放置一些静态资源（图片），注意：public文件夹中的静态资源，webpack打包的时候会原封不动的打包到dist文件夹中

src/asserts文件夹：一般也是放置一些静态资源（一般放置多个组件共用的静态资源），webpack打包的时候会将静态资源当作一个模块，打包到js文件里面

package-lock.json：一个缓存行的文件（可以删除），缓存了依赖从哪个地方下载的，再次下载就比较快了

# 脚手架常用配置

# 项目启动时自动打开浏览器

  1. package.json文件中的serve项添加"--open"参数

    "scripts": {
    "serve": "vue-cli-service serve --open",
  },

  2. npm run serve启动项目

# eslint校验功能关闭

  1. 项目根目录下创建vue.config.js文件

    module.exports = {
    lintOnSave: false
}

  2. 重新启动项目

# src文件夹简写方式，配置别名@

  1. 项目根目录下创建jsconfig.json文件（不配置好像也可以，只是不配置的话@符号是没有提示的？）

    {
    "compilerOptions": {
        "baseUrl": "./",
        "paths": {
            "@/*":["src/*"]
        }
    },
    "exclude": ["node_modules","dist"]
}

#别名@不能在"node_modules"和"dist"文件夹下使用，只能在src文件夹下使用（在src/*.vue文件中才能使用@）

  2. 重新运行项目

# less安装、配置

cnpm install --save less less-loader@6 #默认安装的是最新版本的 less-loader，指定安装的版本即可

查看包都有哪些可选的版本：npm view webpack versions |npm view less-loader version（version不加s好像是查看最新版本）， #不要安装8和9版本，vue2测试的时候安装less-loader@7报错，使用less-loader@6解决
