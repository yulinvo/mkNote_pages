# 起步

python是使用C开发而来

C语言：代码编译得到机器码，机器码在处理器上直接执行，每一条指令控制CPU工作（同样的功能c100行代码，python10行代码 ）

cpython：所说的python就是这种python，是默认情况下的python，使用C解释器执行，.pyc （有pyc文件没有py文件， 程序也可以正常执行（正常运行的顺序是先找pyc文件），只是没法在更改了）

jpython：java实现的python，java解释器执行，解释为java能识别的字节码

ironpython：C#解释器，C#字节码

rubypython、jspython略，同上

pypy：据说一定会火，因为速度快，解释器--字节码--机器码，编译完后直接是机器码（而Cpython的编译过程只是得到了字节码），编译过程虽然慢，但是执行过程就很快了 gli：性能解释器锁，导致运行慢（pandas处理大数据的，重写了gli，速度很快）

python -h    查看帮助，有一个-m的参数，如#python -m json.tool
-	python -m py_compile module.py            #编译成pyc文件
    #python module.pyc                   #执行编译文件，ipython中import可以执行import module，默认引入的就是pyc
-	python -O  -m py_compile module.py         #生成module.pyo的字节码文件，据说执行的比pyc字节码文件快一点
    #引入的方法：python -O 进入python的shell试图(ipython好像不支持-O模式)，然后import module就可以引入pyo(可以先删除py、pyc文件测试)

# 编码

ascii: 8bit

unicode: 16bit 万国码 （python 2.7里面默认是unicode）

utf-8: 字母数字是ascii，其他中文和语言用unicode，实际上是对unicode进行了压缩

utf8转为gbk：utf8-->unicode-->gbk

encode编码：xx --> unicode # xx为gbk、utf8等

decode解码：unicode --> xx # xx为gbk、utf8等

# 深浅拷贝

很多数据类型都有__copy__方法，改方法实际上是调用copy.copy方法

分两类：字符串和数字为一类(字符串池和数字池)，其他的为一类

# 字符串和数字

#无论赋值、深拷贝、浅拷贝内存中使用的地址始终是一样的：

a1=123123
a2=123123                    #id相同（字符串略）
a2=a1                        #id相同（字符串略）
a2=copy.copy(a1)             #id相同（字符串略）
a2=copy.deepcopy(a1)         #id相同（字符串略）
print id(a1);print id(a2)

# 其他：元组、列表、字典

浅拷贝：copy.copy()：浅拷贝只拷贝第一层 深拷贝：copy.deepcopy()：深拷贝有几层就拷贝几层 赋值：拷贝了0层

  • 浅拷贝

    n1={'k1':'wu','k2':123,'k3':['alex',456]}
#n2={'k1':'wu','k2':123,'k3':['alex',456]}

n2=copy.copy(n1)
    print id(n1);print id(n2)      #不同
    print id(n1['k3']);print id(n2['k3'])      #相同，k1-k3都相同
    print id(n1['k3'][1]);print id(n2['k3'][1])      #相同

  • 深拷贝

    n1={'k1':'wu','k2':123,'k3':['alex',456]}
#n2={'k1':'wu','k2':123,'k3':['alex',456]}

n2=copy.deepcopy(n1)
    print id(n1);print id(n2)      #不同
    print id(n1['k3']);print id(n2['k3'])      #k1-k2相同，k3不同
    print id(n1['k3'][1]);print id(n2['k3'][1])      #k3列表里面的值相同

  • 赋值

    n1={'k1':'wu','k2':123,'k3':['alex',456]}
n2={'k1':'wu','k2':123,'k3':['alex',456]}      #不同（有点类似于深拷贝）
n2 == n1      #相同,id不相同
