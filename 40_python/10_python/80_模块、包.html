模块引入 #

模块引入方式：

方式一： 引入整个模块
	import  demo   &&  print(demo.age)   或者  import  demo  as do &&  print(do.age)

方式二： 只引入模块中的部分方法
	 from demo import age 	或者	 from demo import age as age111

#cat aa.py:
    __all__ = ['add',]		# noqa (加上这个好像是语法不检查还是什么)       
        #控制模块对外接口,不是强制性的！！！！
        -	from aa import *只会引入add，可以在ipython中执行dir()查看环境变量中的内容有没有say 
        -	但是from module import  say依然可以引用
        -	但是import aa，依然可以使用module.say()

    def _say():    print('111')		
        #一个下划线开头的函数是模块私有的，不希望被外部调用，但是也不是强制的，外部依然可以调用，君子协定

    def say():    print('say...12')
    def add(a,b):    return a+b

    if __name__ == "__main__":    say()			#直接执行import aa时不会输出‘say...12'


import aa	&&	aa.__xx				#aa.__	使用tab查看都有哪些方法		#直接在ipython中import aa即可！！！
​	#aa.__name__	   	输出str格式结果，'aa'，__name__是一个方法，直接执行__name__输出的是__main__
​	#aa.__file__

包的引入 #

_init_.py文件：当引入一个包的变量或方法时，默认就会先执行__init__.py中代码，比如aa包的init语句中有一个print("引入aa包中的模块...")语句，import aa时就会先输出这一句

_init_.py文件中的变量、函数都属于是包的，比如init文件中写入一个def func1():pass，可以：1，import aa && aa.func1()调用 2，from aa import func1调用 ！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！！

from aa.xx 这种写法，点后面的xx只能是包名（即使在__init__中引入了也不行）【也是，_init_中定义的内容就相当于是包自身的属性了，当然可以直接调取，不用再次导入了】；from aa import xx这样可以直接引入__init__中定义的内容 ！！！！！

from django.http import request #http是包，则request 1：要么是包下面的模块，2：要么是__init__.py中引入的快捷方式！！！

目录结构：
+--- 01.py					#文件内容为空
+--- aa
|   +--- __init__.py		#文件中只有一行代码：	print("引入aa包中的模块...")
|   +--- a1.py				#文件中只有一行代码： name1 = "a1"
|   +--- a2.py				#文件中只有一行代码： name2 = "a2"

  1. 在01.py中引入aa/a2.py中的name2变量，执行的是01.py文件，这个是期望的结果没有问题

    $ cat 01.py
	from aa.a2 import name2			
		#注意这句代码的执行顺序：1，先执行包的__init__.py  2，执行a2中的全部代码  ！！！！！！！！！！！！
		#接上，即使这段代码是写在包的__init__.py文件中的，也会执行a2中的全部代码  ！！！！！！！！！！！！
	print(name2)

# 执行结果为：
	引入aa包中的模块...
	a2

  2. 01.py文件内容和上面的内容保持不变，

    • 在aa/a2.py中引入aa/a1.py的name1变量，当执行的是a2.py文件时正常输出"a1"

      $ cat a2.py

相对引入 #

绝对引入时是通过sys.path查找路径；

相对引入也需要先使用sys.path找到py文件所属于的包（不过要先告诉解释器文件所属于的包是哪个） ！！！！！！！！！！

参考文档：Python 中的绝对导入和相对导入 - 知乎 (zhihu.com)

​ python import中的绝对引用和相对引用_哔哩哔哩_bilibili

目录结构：
+--- aa
|   +--- __init__.py		#文件中只有一行代码：	print("引入aa包中的模块...")
|   +--- a1.py				#文件中只有一行代码： name1 = "a1"
|   +--- a2.py				#文件中只有一行代码： name2 = "a2"

a1.py中代码"from . import a2"执行时报错"ImportError: attempted relative import with no known parent package"；这个跟是否是顶层目录没有关系，aa下在建一层bb，下两个文件b1.py/b2.py结构和内容类似上面，依然是报同样的错误，说明跟目录深度没有关系；但是子包之间通过相对引用时，通过主程序来运行的话是没有问题的（B站参考视频的13.00左右），因为子包被调用时__package__和__name__是有正确的值的，_init_能使用相对路径引入也是这个道理！！！！！！！

a2.py中添加打印语句：
	print('__file__={0:<35} | __name__={1:<20} | __package__={2:<20}'.format(
    __file__, __name__, str(__package__)))

a1.py中添加代码：
	sys.path.append('c:\\Users\\yulinvo\\Desktop\\temp\\python3_1634879321\\code0001')	
		#要先能找到上层包aa
		#sys.path.append只是程序执行时临时加进去，执行结束后就自动释放了，测试：单独在a2.py中打印sys.path就没有这个路径
	
	__package__ = 'aa'			#__name__和__package__使用任意一个即可
		#如果 __package__ 为 None 表示该模块是顶层模块！！！！！！！！！！！！
	# __name__ = 'aa.xx'		#__name__和__package__使用任意一个即可
	from . import a2			#这样就能正常引入了
	
	
a1.py中代码执行后输出结果：
		引入aa包中的模块...
		__file__=c:\Users\...\code0001\aa\a2.py | __name__=aa.a2    | __package__=aa
