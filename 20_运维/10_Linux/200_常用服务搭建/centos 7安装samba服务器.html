  • 环境准备
1，解压源码包，进入目录：samba-3.5.9/source3
2，./configure --prefix=/usr/local/samba-3.5.9   环境配置
        可能在刚开始时，该目录下没有configure文件，此时可先执行下面的命令# ./autogen-sh，此时要求系统安装了autoconf、automake等工具
3，make && make install  编译安装
4，配置动态链接库路径：
     安装完成后，我们需要配置动态链接库路径，因为运行samba的进行smbd和nmbd需要到目录/usr/local/samba/lib下的动态链接库文件。但是该目录不是系统默认的动态链接库文件搜索路径，
        于是我们需要将该目录添加到文件ld.so.conf中。
             # vi /etc/ld.so.conf        添加一行：/usr/local/samba-3.5.9/lib
             然后执行命令  # ldconfig                来更新动态链接库缓冲。
5，在samba-3.5.9/lib下创建smb.conf文件;配置文件smb.conf的默认路径是/usr/local/samba/lib/smb.conf, 但是3.5.9版本的好像没有
[global]
         workgroup = MYGROUP
         server string = Samba Server %v
         security = user
         log file = /var/log/samba/%m.log
         passdb backend = smbpasswd
         username map = /usr/local/samba3.5.9/lib/smbusers
         smb passwd file = /usr/local/samba3.5.9/private/smbpasswd
[root]
         path = /
         valid users = root
         writeable = yes
[public]
         path = /data
         guest ok = yes
         read only = yes

映射：
        创建文件：samba3.5.9/lib/smbusers
        # Unix_name = SMB_name1 SMB_name2 ...
        root = administrator admin
        nobody = guest pcguest smbguest

6，启动samba

      ./testparm /usr/local/samba/lib/smb.conf                检查配置文件是否生效(要先执行ldconfig，不执行会报错）

      ./smbpasswd –a root                修改root用户密码（启动进程后修改也有效）
                向smb.conf文件中指定的/etc/samba/smbpasswd文件中添加root用户以及密码信息
                在使用smbpasswd时默认读取的密码文件时/usr/local/samba/private/smbpasswd，我们可以通过smbpasswd file来指定特定的密码文件。

                smbpasswd -a 增加用户（要增加的用户必须以是系统用户）
                smbpasswd -d 冻结用户，就是这个用户不能在登录了
                smbpasswd -e 恢复用户，解冻用户，让冻结的用户可以在使用
                smbpasswd -n 把用户的密码设置成空.
                     要在global中写入 null passwords -true
                smbpasswd -x  删除用户

        关闭防火墙后启动程序（我当时把/etc/samba文件夹下清空了，就没加-s和后面的参数）：
        # /usr/local/samba/sbin/smbd –D –s /etc/samba/smb.conf
        # /usr/local/samba/sbin/nmbd –D –s /etc/samba/smb.conf
        samba有两个主要的进程smbd和nmbd。smbd进程提供了文件和打印服务，而nmbd则提供了NetBIOS名称服务和浏览支持，帮助SMB客户定位服务器，处理所有基于UDP的协议。

7，测试检查
./smbclient –L //127.0.0.1
格式如下：
[root@localhost bin]# ./smbclient –L //127.0.0.1

当然也可以通过smbclient来访问samba服务器提供的服务，其命令格式如下：
# ./smbclient “//127.0.0.1/root” –U root
