nginx不支持模块化的方式（apache支持），要结合php只能是fastcgi的方式

php-fpm : 127.0.0.1：9000

[root@vmware ~]#netstat -lnp |grep ':80**>' #牟定词尾，这样就过滤掉了8080端口，也可以 ':80\b**'

netstat -ant |awk '/:80>/{S[$NF]++}END{for (A in S) {print A,S[A]}}' #统计80端口连接 各种状态的个数

​ $NF是最后一个字段的名称当成下表，此处就是状态；awk '{S[$NF]++}END{for (A in S) {print A,S[A]}}' 取出所有端口连接状态，不只80端口

yum groupinfo 'Devel ...' #查看组里面的包都有哪些

./configure --prefix=/usr/local/nginx_1.13  --with-pcre

用户对web页面的请求直接到nginx上，只是如果是php页面的的请求则转给php，通过fast-cgi协议转给php，fast-cgi协议也需要监听在某个套接字上；php响应完后再给nginx，由nginx构建http(因为请求者无法理解fastcgi协议)报文返回给请求者

同步模型：nginx收到php返回的fastcgi协议报文直接转发给请求者，nginx只是一个透明转发的功能

异步模型+本地缓存：

​ 异步：nginx收到php返回的fastcgi协议报文，然后封装为http报文返回给请求者

​ 本地缓存：nginx收到php返回的fastcgi协议报文先缓存一份，其他请求者请求内容如果缓存有（没过期）则直接从这里面取，不用交由php处理了，也可将常见的请求，如php主页放到这里

nginx维持1w个非活动连接，仅需要2.5M内存

当连接数很多mysql压力大时，做一个读写分离（即主从服务器），一台专门做读、一台专门写；如果还不够，在加从服务器

​ 还需要加memcache，对速度提升非常非常明显（memcache命中率要高，40%就差不多吧）

​ memcache还可以缓存会话

nginx缓存--->xcache(php编译结果缓存)--->memcache（mysql查询结果缓存）

代理：所谓代理，就是它帮你找 你需要的东西

正向代理：在客户端前端，如上网代理，类似与总管

反向代理：在服务器前端，如服务器不让直接访问，前端加一个反向代理服务器，类似与秘书

​ nginx的反向代理常被用来代理的服务：web、mail（很少被人提到）

fastcgi_pass：正向代理

proxy_pass：反向代理（rewrite很关键）

nginx的启动脚本：

cat /etc/init.d/nginx

#! /bin/bash
# chkconfig: 35 85 15  
# description: Nginx is an HTTP(S) server, HTTP(S) reverse
set -e
DESC="nginx daemon"
NAME=nginx
DAEMON=/usr/local/nginx-1.10/sbin/$NAME
SCRIPTNAME=/etc/init.d/$NAME
test -x $DAEMON || exit 0
d_start(){
    $DAEMON || echo -n " already running"
}
d_stop() {
    $DAEMON -s quit || echo -n " not running"
}
d_reload() {
    $DAEMON -s reload || echo -n " counld not reload" && echo &&exit
}
case "$1" in
start)
    echo -n "Starting $DESC:$NAME"
    d_start
    echo "."
;;
stop)
    echo -n "Stopping $DESC:$NAME"
    d_stop
    echo "."
;;
reload)
    echo -n "Reloading $DESC configuration..."
    d_reload
    echo "reloaded."
;;
restart)
    echo -n "Restarting $DESC: $NAME"
    d_stop
    sleep 2
    d_start
    echo "."
;;
*)
    echo "Usage: $SCRIPTNAME {start|stop|restart|reload}" >&2
    exit 3
;;
esac
exit 0

location格式：
    location [ = | ~ | ~* | ^~ ] uri {...}
    location { } @name {...}

location  uri { } :      #｛｝定义的属性对uri起始路径及子路径下的所有对象都生效
location  = uri { } :  #｛｝定义的属性只对当前路径生效；如果uri是一个目录则只对目录访问生效，对目录下的文件不生效
location  ~ uri { } :    #模式匹配，此处的uri可以使用正则表达式，区分字符大小写
location  ~* uri { } :    #模式匹配，同~，唯一的区别是不区分字符大小写
location  ^~ uri { } :     #不使用（禁用）正则表达式，逐字符匹配
如果被多个匹配到，生效顺序（优先级）：
        =最高 > ^~次之  > ~（~*）次之 >没有任何符号的
      /document/document时         location  /document { } :比 location  / { } :更精确，先比配到这个

重定向：

临时重定向：302
last：本次重写结束了，重启下一轮检查        #通常使用的都是last
       如rewrite '^/bbs/(.*)'     http://1.1.1.1/forum,重写后还要检查/forum 
break：本次重写结束了，不再重启下一轮检查，直接执行后续操作
redirect：临时重定向
permanent：永久重定向
