./configure \
--prefix=/usr/local/apache \
--enable-so \
--enable-ssl \
--enable-cgi \
--enable-cgid   \
--enable-rewrite \
--with-zlib \
--with-pcre \
--with-apr=/usr/local/apr \
--with-apr-util=/usr/local/apr-util \
--enable-modules=most \
--enable-mods-shared=most \
--enable-mpms-shared=all    
(--with-mpm=event )        #默认就是event，这一句不用指定

#--with-apr=/usr/local/apr --with-apr-util=/usr/local/apr-util 可以使用 --with-included-apr 代替
--sysconfdir    指定的是配置文件的安装路径
--enable-so    支持动态共享模块
--enable-mods-shared    ：是否启用共享模块
mpm：prefork、worker、event（后两种需要php编译成zts格式）
        httpd v2.4可以支持3种格式切换，且v2.4的默认支持的是event格式的（性能最好），之前的版本rpm安装默认是prefork模式的
        --enable-mpms-shared=，    可以指定使用三个中的哪些，或者是all
        --with-mpm=      指定使用三个中的哪个作为默认值，默认就是event；
                                        如果没有--enable-mpms-shared只有--with-mpm=这句，表示只安装event

--enable-modules=most  会安装一个2.4才有的模块，mod_proxy_fcgi ，这个模块如果之前版本要使用，需要额外编译

mpm:

prefork：（一个请求用一个进程响应）预先生成进程
worker:（一个请求用一个线程响应）基于线程来工作；一个进程相应多个用户请求，但是一个进程下使用多个线程相应用户请求
event:（一个进程处理多个请求）基于事件的驱动，一个进程处理多个用户请求，不是基于线程而是一个进程直接处理多个请求

apr：不同的平台需要不同的httpd版本，一个折中的方法是通过apr实现不同系统平台的移植，类似与java的虚拟机

includes 头文件

apache安装完后目录：

build：编译的时候一个目录，不用管它
cgi-bin：执行cgi程序的存放位置

manual：手册，官方手册
modules：模块目录

遇见问题总结：

问题：You don't have permission to access /index.html on this server.

原因：
开始以为是因为权限的配置问题，将 Require all denied改为 Require all granted之后问题依然存在
<Directory />
    AllowOverride none
    Require all denied
</Directory>


可能存在的原因，这边遇见的问题是第二种原因造成的：
            原因1：index.html是用root用户建的文件，apache权限不够。
            解决方法：更改文件权限；chmod 755 index.html
    原因2：是Apache  的配置文件httpd-vhosts.conf里，打开了扩展配置
            # Virtual hosts
           Include conf/extra/httpd-vhosts.conf
            前面的#去掉了，就 引入了扩展配置，而扩展配置 文件里又没配好相应 的权限“Allow from all”，所以提示此错误

apache的查看帮助信息：

httpd.conf中打开：Include conf/extra/httpd-manual.conf
输入：http://192.168.86.101/manual/  即可查看帮助信息

[root@vm-2 /usr/local/apache/conf]#httpd -t
AH00526: Syntax error on line 36 of /usr/local/apache/conf/extra/httpd-manual.conf:
Invalid command 'LanguagePriority', perhaps misspelled or defined by a module not included in the server configuration
解决方法：
    httpd.conf中打开LoadModule negotiation_module modules/mod_negotiation.so

另外：httpd-manual.conf中第36行将语言优先次序改下好像有用
        LanguagePriority  zh-cn  en  fr  ko ja tr es de  pt-br da ru
